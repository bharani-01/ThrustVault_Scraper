# Security Policy

## Supported Versions

| Version | Supported | Security Patches |
|---------|-----------|-----------------|
| 2.0.x   | ✅ Active  | Yes             |
| 1.x.x   | ⚠️ Legacy  | Critical only   |
| < 1.0   | ❌ EOL     | No              |

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

To report a security vulnerability privately:

1. Navigate to the [GitHub Security Advisories](https://github.com/bharani-01/ThrustVault_Scraper/security/advisories/new) page.
2. Click **"Report a vulnerability"**.
3. Fill in a detailed description including:
   - Nature of the vulnerability (e.g., injection, path traversal, credential exposure)
   - Reproduction steps (minimal example preferred)
   - Potential impact (data exposure, denial of service, etc.)
   - Suggested fix (if known)

**Response SLA:**
- Acknowledgement: within **48 hours**
- Full assessment: within **7 business days**
- Fix release: within **30 days** for critical issues

We practice **coordinated disclosure** — vulnerabilities will not be disclosed publicly until a patch is released or a reasonable remediation timeline has passed.
